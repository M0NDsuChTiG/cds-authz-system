Docker Zero-Trust Authorization System

Key Features

Managed Trust Store

Securely store and manage public keys with versioning and full audit tracking for every administrative action.

Fail-Closed Policy

If the daemon is unreachable or verification fails, container launches are denied by default. Maximum security.

Cosign Integration

Native support for Sigstore/Cosign ensures that only cryptographically signed images can run in your cluster.

Audit Streaming

Real-time logging of allow/deny decisions to syslog and local BoltDB for forensic analysis and monitoring.

Quick Start

Get up and running with CDS v0.6.1 in minutes.

# 1. Install via the automated script 
git clone https://github.com/M0NDsuChTiG/cds-authz-system.git
cd cds-authz-system
./install.sh
# 2. Import your production public key 
cds-cli key import --id prod --path ./my-key.pub
# 3. Add trust for a specific image digest 
cds-cli trust add alpine:latest --key-id prod --digest sha256:251091...
# 4. Verify status 
cds-cli trust list

Security Posture

CDS is designed with defense-in-depth principles:

  • Systemd Sandboxing: Services run with ProtectSystem=strict and NoNewPrivileges=true.
  • UNIX Socket Isolation: Communication happens over restricted local sockets with strict permissions.
  • Immutable Trust: Trust decisions are bound to image digests (sha256), preventing tag-swapping attacks.
